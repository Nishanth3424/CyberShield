CYBERSHIELD — Operational Briefing

What is CyberShield?

CyberShield is a live network intrusion detection visualizer. It simulates a small corporate network — gateway, firewall, router, servers, and workstations — with animated packet flows between them. You play the role of an attacker, launching real attack patterns: DDoS floods, port scans, MITM hijacks, SSH brute-force attempts, and data exfiltration. The built-in IDS detects them in real time, raises classified alerts, and a built-in analyst explains why each alert fired.

DDoS Flood

Mass packet burst toward the web server. Caught by rate-based detection — packets-per-second crossing the threshold trigger escalating severity.

Port Scan

Sequential probing of TCP ports. Caught by sequential pattern detection in a sliding window of recent destination ports.

MITM Hijack

Traffic rerouted through an attacker node. Caught by routing anomaly detection — packets traversing unexpected hops.

Brute Force

Repeated SSH auth attempts on one workstation. Caught by protocol concentration analysis on port 22.

Data Exfil

Anomalously large outbound transfers from the DB server. Caught by egress volume analysis.

Ransomware

Lateral spread between workstations/servers mimicking WannaCry. Caught by internal lateral movement pattern detection.

SQL Injection

Malicious queries targeting DB port 3306 from external sources. Caught by protocol-specific query rate analysis.

DNS Tunneling

Covert data channel via oversized DNS packets. Caught by DNS payload size and frequency anomaly detection.

Health Score

Starts at 100. Critical alerts drop it by 5, warnings by 1. Regenerates +0.4/tick in normal mode. Visualizes overall security posture.

How To Use

1. Watch the network — packets stream between nodes color-coded by protocol (HTTP cyan, HTTPS green, DNS violet, SSH amber, SQL pink).

2. Click an attack button (left panel). The attacker node animates in and floods the network with malicious traffic.

3. Watch the alert feed populate as the IDS detects the attack — color-coded by severity (info/warning/critical).

4. Hit Run Threat Analysis for an automated written explanation of the current network state, top threat sources, and recommended mitigations.

5. Switch back to NORMAL mode and watch the health score regenerate.

Detection Algorithms (Implemented From Scratch)

Rate-based: Sliding 2.5s window counts packets per node. Severity scales with how far above threshold the rate climbs (1.5× = warning, 3× = critical).

Sequential port: Ring buffer of last 20 destination ports per node. Slides a window of 5 looking for ascending consecutive values.

Routing anomaly: Each packet carries an expected path. If actual path traverses an unexpected node (the attacker), it's flagged critical.

Protocol concentration: If >70% of recent packets to a node are the same protocol/port, that's suspicious (brute-force signature).

Egress volume: Tracks bytes-per-tick leaving each node. Anomalous spikes from internal nodes = data exfiltration indicator.

Deduplication: Same (alert-type, node) pair is suppressed for 2 seconds after firing — prevents alert fatigue during sustained attacks.

Lateral movement: Tracks internal-to-internal malicious traffic patterns. Multiple payloads arriving at the same host from internal sources triggers ransomware alerts.

SQL injection: Monitors port 3306 query concentration from external sources. Rapid sequential queries trigger critical alerts.

DNS tunneling: Tracks DNS payload volume and frequency. Abnormally large DNS packets and high query rates indicate covert data channels.

Why This Project?

Intrusion detection is everywhere in modern security — every firewall, every cloud workload, every SOC dashboard runs algorithms exactly like these. CyberShield turns a normally invisible, log-only discipline into something you can watch happen. Trigger a DDoS, see packets pile up, watch the alert feed fire, see the health bar drop — the entire detection loop, visually, in one screen.
